## 下一步优化方案（尚未实施）

### Phase 4：前端体验优化（优先级：高） ✅ 已完成

#### 4.1 PDF Worker 本地化 ✅
- **问题**：ReaderPage.tsx 从 unpkg CDN 加载 pdf.worker.js，网络不可用时白屏
- **方案**：将 pdf.worker.js 复制到 `public/` 目录，修改 workerSrc 为本地路径
- **文件**：`src/modules/reader/ReaderPage.tsx:16`, `public/pdf.worker.min.mjs`

#### 4.2 ReaderPage 实现空壳功能 ✅
- **问题**：全屏按钮和批注按钮 onClick 未实现
- **方案**：
  - 全屏：使用 `document.documentElement.requestFullscreen()` API + fullscreenchange 事件监听
  - 批注：文本选中→调用 addAnnotation IPC 保存高亮，按钮支持 toggle 高亮模式
- **文件**：`src/modules/reader/ReaderPage.tsx`

#### 4.3 GraphPage Canvas 动态尺寸 ✅
- **问题**：Canvas 固定 900x440，CSS 100% 压缩导致变形
- **方案**：使用 ResizeObserver 监听容器尺寸，动态设置 canvas width/height 属性，节点布局坐标基于canvas尺寸计算
- **文件**：`src/modules/graph/GraphPage.tsx`

#### 4.4 AI 错误消息视觉区分 ✅
- **问题**：AI 回复失败时消息样式与正常回复一致，用户无法区分
- **方案**：错误消息添加红色边框 + WarningOutlined 图标 + "失败" Tag
- **文件**：`src/modules/ai/AIPage.tsx`, `src/modules/reader/ReaderPage.tsx`

#### 4.5 AI Provider 列表不一致 ✅（已存在）
- **问题**：AIPage 的 AI_PROVIDERS 列表缺少 ollama，但默认 provider 是 ollama
- **现状**：AI_PROVIDERS 列表已包含 ollama，无需修改

### Phase 5：性能深度优化（优先级：中）

#### 5.1 httpx AsyncClient
- **问题**：embedder.py 使用同步 httpx.Client，在 async FastAPI 中阻塞事件循环
- **方案**：改用 `httpx.AsyncClient`，将 `embed` / `_embed_openai_compat` 改为 async
- **影响**：需要将 vectorstore.py 中的嵌入调用也改为 async
- **文件**：`sidecar/app/services/embedder.py`

#### 5.2 Zotero 导入 N+1 优化
- **问题**：每条论文 5 次独立 SQL 查询（fields, creators, tags, collections, attachment）
- **方案**：使用 JOIN 一次性获取所有数据，或批量 IN 查询
- **文件**：`electron/services/zotero-import.ts:206-272`

#### 5.3 fetchPapers 竞态保护
- **问题**：快速切换筛选条件时，多个 fetchPapers 并发运行，结果乱序
- **方案**：在 useLibraryStore 中添加请求序列号，只接受最新请求的结果
- **文件**：`src/stores/useLibraryStore.ts:45-61`

#### 5.4 RAG 批量摄取并发
- **问题**：批量摄取串行执行每个文档
- **方案**：使用 `asyncio.gather()` + `asyncio.Semaphore(3)` 限流并发
- **文件**：`sidecar/app/routers/rag.py:104-121`

#### 5.5 RAG 查询结果批量获取
- **问题**：每个 chunk_id 单独调用 get_chunk（N+1）
- **方案**：添加 `get_chunks_batch(chunk_ids: list[str])` 方法，使用 `WHERE chunk_id IN (...)`
- **文件**：`sidecar/app/services/vectorstore.py` + `sidecar/app/routers/rag.py:159-173`

### Phase 6：架构打磨（优先级：低）

#### 6.1 electron.d.ts 类型修正
- **问题**：类型定义与 preload 实际接口不匹配（invoke 泛型 vs 具体方法）
- **方案**：移除泛型 invoke，只保留具体类型方法；窗口控制用 `minimizeWindow()` 等
- **文件**：`src/types/electron.d.ts`

#### 6.2 统一错误处理模式
- **问题**：前端几乎所有 async 操作只有 console.error，无用户可见反馈
- **方案**：创建统一的 `handleApiError(err, message)` 工具函数，自动显示 message.error
- **影响范围**：所有 store 和页面组件

#### 6.3 settings 页面"测试连接"
- **问题**：AI/RAG 配置填写后无法即时验证
- **方案**：添加"测试连接"按钮，调用 health check 端点验证
- **文件**：`src/modules/settings/SettingsPage.tsx`

#### 6.4 硬编码常量集中配置
- **问题**：超时、分页大小、默认模型等散落在各文件
- **方案**：创建 `src/config/constants.ts` 集中管理
- **涉及常量**：AI 默认模型、分页 pageSize (20)、搜索 topK (8)、超时时间等

#### 6.5 ReviewPage AI 输出解析加固
- **问题**：AI 输出格式解析脆弱（依赖 `|` 分隔符）
- **方案**：改用 JSON 格式提示 + 容错解析（尝试 JSON，fallback 到 `|` 分隔）
- **文件**：`src/modules/review/ReviewPage.tsx:74-85`

#### 6.6 配置值验证
- **问题**：Python config.py 无 __post_init__ 验证
- **方案**：添加 `__post_init__` 校验 embedding_dim > 0, chunk_size > 0 等关键参数
- **文件**：`sidecar/app/config.py`

#### 6.7 添加 CI 依赖安全扫描
- **问题**：package.json 依赖多，无 npm audit
- **方案**：CI 流程中添加 `npm audit` 步骤
- **文件**：`.github/workflows/build.yml`

## 验收路线图

| Phase | 预计时间 | 完成后验收项 |
|-------|---------|-------------|
| Phase 4（前端体验） | 2-3 天 | PDF 加载可靠性、全屏/批注可用、图谱不变形、AI 错误可见 |
| Phase 5（性能深度） | 2-3 天 | Zotero 大库导入速度、搜索并发安全、RAG 批量摄取吞吐 |
| Phase 6（架构打磨） | 1-2 天 | 类型安全、统一错误处理、配置集中化 |

### 最终验收标准

1. **安全性**：无严重安全问题（SSRF、注入、信息泄露）
2. **数据完整性**：所有写操作有事务保护，删除有级联
3. **性能**：N+1 查询消除，批量操作并发
4. **用户体验**：错误可见可操作，空壳功能补齐
5. **构建部署**：Windows/Linux 可构建，macOS 可恢复
